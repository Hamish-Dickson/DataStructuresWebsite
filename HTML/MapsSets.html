Overview of Maps and Keys

Maps and Keys are, in my opinion, the most complicated data structure I will be covering on this site.

They effectively map a value to another value, and can have different data types for the map and the key.

I utilised a HashMap in my project last year to map error codes to error messages in my room booking system.

Example

I will use the aforementioned example from my project to explain maps and keys.

I had the Error codes 1 through 5, each corresponding to a different message depending on the error that occurred.

I used a HashMap mapping Integers to Strings to store this data. On the following page I will show you how I implemented this.

Example in Java

Initialising the HashMap

pretty simple declaration with the syntax being

HashMap<KeyType, DataType>hashMapName = new HashMap<>();

Note that as with the lists syntax, the second angle brackets do not require information, as it is inferred from the left hand side of the declaration.
